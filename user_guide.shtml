Introduction

This guide corresponds to version 0.2 of pythonizer.

this is alpha version, so do not expect any perfection. It woks but with errors.  It still has bugs and not all constructs are transliterate correctly.

Also detecting of the type of variables is not implemented and that increase the number of errors in Python as conversion to string is not automatic like in Perl.

Here is example of the protocol  Full protocol of translation of pre_pythonizer.pl by version 0.07 of pythonizer

In crease chances of correct transliteration is is recommended to run the Perl script via pre_pythonizer.pl

Parts that can't be translated during the first invocation can be commented outs and iteratively the stage can be reached when the Perl script is completely Pythonized.

Structure

Pythonizer consists of the main program called, as you can guess, pythonizer and three modules

  • Perlscan.pm
  • Pythonizer.pm
  • Softpano.pm

The total size of the codebase in version 0.2 is around 2K lines 

wc -l Perlscan.pm Pythonizer.pm  pythonizer.pl  Softpano.pm
  659 Perlscan.pm
  275 Pythonizer.pm
  948 pythonizer.pl
  249 Softpano.pm
 2131 total

Installation

Currently main program and all modules should reside in a single directory from which you will run the program.  During invocation of pythonizer this directory should be current.

Invocation

You can run Pythonizer both in Cygwin and Linux.

pythonizer -h provides a  list of options

To "pythonize" the script /path/to/your/program  you need to use the following invocation (the directory in which pythonizer resides should be current otherwise modules will not be loaded) 

cd /path/to/pythonizer && pythonizer /path/to/your/program.pl

If the program runs to the end you will get "pythonized" text in /path/to/your/program.py

It also produces protocol of translation as it goes which allot to detect places that need to be commented out translated manually.
